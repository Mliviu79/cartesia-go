# Contributing

This SDK is maintained for internal use. Contributions are welcome but reviewed on a case-by-case basis.

## Getting Started

```bash
git clone https://github.com/Mliviu79/cartesia-go.git
cd cartesia-go
go mod download
```

## Development

### Build

```bash
go build ./...
```

### Test

```bash
go test -race -count=1 ./...
```

### Vet

```bash
go vet ./...
```

## Guidelines

- Follow standard Go conventions (`gofmt`, `go vet`, `golint`)
- All exported types and functions must have doc comments
- All methods accept `context.Context` as the first parameter
- Use table-driven tests
- Run tests with `-race` flag
- Keep files under 300 lines where practical
- Use `errors.As` / `errors.Is` for error checking
- No `interface{}` -- use `any`

## File Organization

```
cartesia-go/
  doc.go              # Package doc, constants
  client.go           # Client struct, NewClient, service init
  request.go          # HTTP execution, retry, tracing, logging
  errors.go           # Error types and helpers
  options.go          # ClientOption functions
  pagination.go       # CursorPage, ListParams, PageIterator
  multipart.go        # Multipart form builder
  sse.go              # SSE event reader
  websocket.go        # WebSocket TTS client
  <resource>.go       # Types + service for each API resource
  <resource>_test.go  # Tests for each resource
```

## Adding a New Endpoint

1. Add types (request params, response) to the appropriate resource file
2. Add the service method using the client's internal request helpers:
   - `requestJSON` -- JSON body, JSON response
   - `requestJSONQuery` -- JSON body with URL query params
   - `requestMultipart` -- multipart form upload
   - `requestDrain` -- fire-and-forget (DELETE, etc.)
3. Add tests using `httptest.NewServer`
4. Run `go build ./... && go test -race ./... && go vet ./...`
